# 运维手册

## 1. 适用范围

这份文档面向日常维护这个系统的人，重点覆盖：

- 服务启动与重启
- 健康检查与日志排查
- 备份与恢复
- 数据库配置与切换
- 常见 AI 与登录问题排查

---

## 2. 服务管理

### 启动或更新整套服务

```bash
docker-compose up -d --build
```

### 只更新后端

```bash
docker-compose up -d --build backend
```

### 只更新前端

```bash
docker-compose up -d --build frontend
```

### 查看容器状态

```bash
docker-compose ps
```

常见状态说明：

- `healthy`：健康检查通过
- `starting`：容器刚启动，仍在等待健康检查
- `unhealthy`：服务已启动，但健康检查失败

---

## 3. 健康检查

系统统一健康检查地址：

```text
http://127.0.0.1:10089/api/health
```

常见用途：

- 发布后快速确认后端是否可用
- 反向代理或容器编排的探活接口
- 排查“页面打开了但接口不可用”问题

如果健康检查失败，优先看后端日志。

---

## 4. 日志排查

### 查看后端日志

```bash
docker logs runtao-academy-backend
```

### 查看前端日志

```bash
docker logs runtao-academy-frontend
```

重点关注：

- `401 / 403`
- `429`
- `500`
- AI 路由超时或模型返回格式错误
- 数据库连接失败

常见判断方式：

- 前端页面报错，但后端日志没有请求：通常是浏览器缓存、前端资源或反向代理问题
- 前端页面报错，后端有 `4xx`：多半是权限、参数或限流问题
- 后端有 `5xx`：优先按堆栈信息定位代码问题

---

## 5. 备份与恢复

备份与恢复是管理员专属功能。

### 导出备份

入口：

- 系统设置页
- `系统管理` 标签下的备份区域

导出结果通常包含：

- 用户
- 分类
- 题目
- 学习进度
- AI 配置
- 系统设置

建议：

- 每次大批量导入、批量删题、数据库切换前先导出一次
- 保留多份历史备份，不要只留最新一份

### 恢复备份

入口同样在系统设置页。

恢复前建议：

1. 先导出当前环境作为回滚点
2. 确认恢复文件来自可信环境
3. 在低峰期操作

恢复后建议检查：

- 能否正常登录
- 题库数量是否符合预期
- 分类和标签是否完整
- AI 配置是否仍可用

---

## 6. 数据库管理

系统支持：

- SQLite
- MySQL

### SQLite

特点：

- 配置简单
- 适合单机和轻量部署
- 默认数据路径在容器内 `/app/data/runtao-academy.db`

### MySQL

适合：

- 多用户协作
- 更正式的生产部署
- 需要独立数据库运维策略的场景

### 数据库配置能力

系统设置页目前支持：

- 新建数据库配置
- 测试连接
- 初始化数据库
- 执行迁移
- 数据校验
- 切换当前运行数据库

建议切库流程：

1. 导出当前备份
2. 新建目标数据库配置
3. 先测试连接
4. 初始化和迁移目标库
5. 做数据校验
6. 再切换运行库

## 7. AI 功能排查

### AI 功能不可用

优先检查：

- 当前用户是否有 `ai_use`
- 具体功能权限是否存在，例如 `ai_generate`、`ai_polish`
- 当前激活的模型配置是否可用
- 对应 API Key 是否存在
- 后端日志是否有模型调用失败

### AI 润色很慢

当前已支持：

- `轻润色`
- `深润色`

排查建议：

- 优先尝试 `轻润色`
- 检查模型本身是否响应偏慢
- 查看后端日志里单次请求耗时

### 自定义 AI 地址相关问题

当前安全策略：

- 普通用户可以使用已有的安全自定义模型
- 普通用户不能新增或修改自定义 `baseUrl`
- 管理员才能管理自定义地址

如果报自定义地址错误，优先检查：

- 地址是否为 `https`
- 是否命中了内网、`localhost`、保留地址等安全限制

## 8. 登录与权限排查

### 登录太频繁

系统对登录和注册有单独限流。

如果提示登录太频繁：

- 等待提示中的秒数或分钟数
- 检查是否有脚本或浏览器重复提交登录
- 查看是否存在异常代理或自动重试

### 登录后立刻掉线

当前系统优先使用 `HttpOnly Cookie` 保存登录态。

优先检查：

- 浏览器是否禁用了 Cookie
- 站点协议和域名是否一致
- 前后端是否被不同域名或代理拆开

### 看得到页面但功能按钮灰掉

通常是权限问题。优先检查：

- 角色是否正确
- 该用户是否具备对应权限
- 如果是集成用户，分类范围是否覆盖了当前题目

---

## 9. 发布检查清单

每次发布后，至少检查这几项：

1. `docker-compose ps`
2. `/api/health`
3. 管理员登录
4. 题库列表能否打开
5. 新建题目是否正常
6. `AI答案`
7. `AI润色`
8. `AI批量生题`
9. 备份导出

---

## 10. 常见问题

### 页面明明重启了，但功能还是旧的

通常是浏览器缓存。先强制刷新：

- Windows/Linux：`Ctrl + F5`
- Mac：`Cmd + Shift + R`

### 后端健康检查正常，但前端样式或按钮不对

优先检查：

- 前端容器是否也已更新
- 页面是否仍在加载旧的 JS 资源
- 浏览器缓存是否命中了旧 bundle

### 数据恢复后权限不对

优先检查：

- 恢复文件里的用户权限数据
- 当前登录用户是否已重新登录刷新权限状态
- 是否切换到了正确的数据库配置
